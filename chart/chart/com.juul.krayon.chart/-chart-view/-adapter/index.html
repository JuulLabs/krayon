Adapter for the ChartView. Ensures that Renderer and DataSet types line up, and handles rendering on background threads in response to various changes. Each Adapter should only be used for a single ChartView.

If a render is under way when a state change is requested, the current render will be canceled and a new one will start. This means the current approach is not particularly animation friendly. In the future I'll see if I can improve this.
